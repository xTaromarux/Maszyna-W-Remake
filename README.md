## 🔧 Konfiguracja platform

Projekt obsługuje warunkowe zachowanie w zależności od platformy docelowej:

### Dostępne opcje:
- `web` – standardowe użycie w przeglądarce  
- `esp` – deployment na ESP32

### Konfiguracja:

**Plik `.env` (web version):**
```env
VITE_APP_PLATFORM=web
VITE_WEBSOCKET_URL=ws://localhost:8080
VITE_HF_PROXY_URL=http://localhost:3001
```

**Plik `.env.esp` (ESP32 version):**
```env
VITE_APP_PLATFORM=esp
VITE_WEBSOCKET_URL=ws://192.168.4.1:8080
VITE_HF_PROXY_URL=disabled
```

### Kompilacja:
```bash
# Wersja web
npm run build

# Wersja ESP32  
npm run build:esp

# Development z ESP32
npm run dev:esp
```

### ESP32 Setup
```cpp
// Przykład kodu Arduino dla ESP32
#include <WiFi.h>
#include <WebSocketsServer.h>

WebSocketsServer webSocket = WebSocketsServer(8080);

void setup() {
  WiFi.softAP("MaszynaW", "password123");
  webSocket.begin();
  webSocket.onEvent(webSocketEvent);
}

void webSocketEvent(uint8_t num, WStype_t type, uint8_t * payload, size_t length) {
  if(type == WStype_TEXT) {
    // Obsługa sygnałów z symulatora
    String message = String((char*)payload);
    handleSignal(message);
  }
}
```
